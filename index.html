Research Motivation

Addressing the reproducibility crisis in biomedical machine learning through architectural guarantees rather than procedural guidelines.

Computational workflows in genomics, clinical prediction, and drug discovery routinely fail to reproduce across labs and time horizons (Baker, 2016; Hutson, 2018). The causes are infrastructural: uncontrolled analytical drift, implicit state dependencies, and the absence of structured evaluation frameworks. An estimated 70% of computational biology results fail to reproduce when re-executed by independent groups. This problem is particularly acute in multi-modal biomedical settings where graph-based reasoning over biological networks, clinical tabular data, molecular structure, and imaging must be combined under a single analytical framework with consistent provenance.

SNPTX addresses these root causes architecturally:

Root CauseArchitectural Response
Analytical driftExtensions are downstream-only, consuming immutable artifacts
Implicit stateAll configuration is explicit; DAG ordering is deterministic
Unstructured comparisonStandardized evaluation artifacts enable systematic model comparison
Missing provenanceArtifact versioning with input hashing, commit embedding, and experiment logging

Separation of execution (what runs and when) from interpretation (how results are analyzed and compared), enabling independent evolution of model architectures, evaluation methods, and feedback strategies.

Extension System

Tier-1 extensions form the analytical surface for downstream interpretation. All extensions are strictly downstream of the core pipeline, consume and produce artifacts only (no shared state), declare explicit typed input/output contracts, and produce deterministic outputs. Extensions live in a physically separate repository, isolating analytical contributions from orchestration logic.

Validation Methodology (8-Check Audit)

Every modality result is validated through 8 independent statistical checks, ensuring reported performance is robust, calibrated, and not an artifact of favorable data splits.

1. k-Fold Cross-Validation

5-fold stratified CV with per-fold metrics and variance reporting. Ensures performance is not an artifact of a single favorable split.

2. Multi-Seed Stability

5 independent random seeds, confirming accuracy variance < 1.2% across runs. Ensures insensitivity to initialization.

3. Bootstrap CIs

1,000-sample bootstrap on test predictions producing 95% confidence intervals on accuracy, F1, and AUC.

4. Spatial Stratification

For tissue data (Visium), splits prevent spatial autocorrelation from inflating performance. Adjacent spots never split across train/test.

5. Fusion Holdout

PCA embeddings extracted from training set only. Test set projected using frozen training-set PCA parameters. No data leakage.

6. Conformal Prediction

Distribution-free prediction sets with coverage guarantees (Vovk et al., 2005). Per-sample uncertainty without distributional assumptions.

7. Learning Curves

Training set subsampled at [0.1, 0.25, 0.5, 0.75, 1.0] fractions to confirm models are not in data-starved regime.

8. Trivial Baselines

Every model compared against majority-class and random baselines. All results significantly exceed trivial baselines.

Reproducibility Controls

Deterministic Guarantees

Fixed seeds: Global seed (1337) for all random operations (numpy, torch, sklearn).

Deterministic CUDA: torch.use_deterministic_algorithms(True) where supported.

Versioned configuration: Every run defined by YAML committed to version control.

DAG-enforced ordering: Snakemake guarantees execution order; no implicit dependencies.

Artifact hashing: DVC checksums verify data and model artifact integrity.

Cross-Modal Benefit

Fusion achieves 93.0% validation accuracy on 14-class tissue region classification, exceeding the best unimodal result (omics VAE at 92.8%) by 0.2 percentage points. While the improvement is modest, it demonstrates that complementary morphological signal from PCA vision features provides additional discriminative power beyond gene expression alone. Holdout accuracy (93.0%) is consistent with 5-fold CV (92.8% ± 0.7%), confirming no leakage.

Closed-Loop Process

The autonomous engine operates as a closed-loop system that: selects experiment configurations using the intelligence layer (adaptive defaults + meta-features), executes training and evaluation through the standard Snakemake pipeline, evaluates results against the experiment catalog history, classifies discoveries by type (new_best, novel_config, surprise), updates the feedback loop with Bayesian confidence scores, and iterates until the campaign budget is exhausted.

Pilot Deployment Configurations

Four configurations available for academic research labs working at the intersection of biomedical data science, drug discovery, and clinical machine learning.

A Drug Discovery Pipeline

GCN-based drug discovery pipeline with knowledge graph integration on lab-specific compound libraries.

Requirements: Molecular activity data (SMILES + labels), or access to ChEMBL/PubChem subsets.

Delivers: Trained GCN model with bioactivity predictions, validation report (5-fold CV, bootstrap CIs, calibration analysis), artifact-versioned pipeline.

Demonstrated: 97.5% accuracy on ChEMBL bioactivity (binary), 97.2% ± 0.6% 5-fold CV.

B Multi-Modal Clinical Research

Multi-modal fusion pipeline combining clinical, omics, imaging, and/or text modalities.

Requirements: At least 2 modalities of patient-level or sample-level data with shared identifiers.

Delivers: Per-modality baseline models, attention-based fusion model, fusion holdout validation (leakage-free), uncertainty quantification with conformal prediction sets.

Demonstrated: 93.0% fusion accuracy (val), integrating omics (92.8%), vision PCA, and spatial features on Visium breast tissue.

C Autonomous Experimentation

Self-learning engine deployed on lab-specific datasets to systematically explore model configurations.

Requirements: One or more supervised learning datasets (any modality SNPTX supports).

Delivers: Autonomous campaign results, intelligence layer outputs (meta-analysis, adaptive defaults), experiment catalog (DuckDB) for post-hoc analysis.

Demonstrated: 1,100 experiments in 1.06 hours, 104 discoveries across 5 datasets and 6 model types.

D Cross-Institutional Reproducibility

Bit-identical reproducibility of a reference pipeline across two compute environments.

Requirements: Two environments with Python 3.11+ and GPU access.

Delivers: Identical outputs confirmed across environments (metrics, artifacts, model weights), reproducibility report with artifact hash comparison.

What SNPTX Does Not Provide

Not a medical device. SNPTX is a research framework; results require expert interpretation.

No pre-trained models for clinical deployment. Models are trained from scratch on provided data.

Data remains on lab infrastructure. SNPTX processes data in-place; no data hosting or transfer.

Design Philosophy

Where results are strong (imaging at 99.3%, drug discovery at 97.5%, single-cell at 96.9%), we report them. Where results are limited by data (clinical text at 61.0%), we report them equally and explain why. The value of SNPTX is not in any single accuracy number but in the infrastructure that makes every result independently verifiable, every comparison fair, and every experiment reproducible.

Embedding Registry

SNPTX maintains a versioned embedding registry: an artifact-driven store of learned representations linked to specific model versions, datasets, and training configurations. This enables downstream extensions to consume representations as stable artifacts, cross-model embedding comparison through standardized interfaces, and transfer learning workflows with full provenance.
